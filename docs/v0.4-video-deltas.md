# v0.4 video-number deltas — old vs new

Hand-off for Video #1 number re-verification (SPEC v0.4 §6 / §7 / §9 step 7).

**Bottom line**: the only cells that move are the ones where v0.3's additive-total gate failed at low channel counts. **T1 always moves** for any cost with both Energy ≥ 1 and Power ≥ 1. **T2 onward almost always agrees** — because by T2, channel count has cleared both the old additive gate and the new energy gate, and the multivariate engine (unchanged) computes identical answers.

If a video number is at T2 or later, it is almost certainly unchanged. If a video number is at T1 — or at any turn where v0.3 returned 0 for what should've been a possible cast — it changed.

## How to use this doc

1. For each number that appears on screen in Video #1, look up the cost / deck / turn / order combo above (or recompute in the running tool — `runes.witchtilt.com` is live on this branch's preview).
2. If the cell is in the **bold** column (T1 with Energy ≥ 1 and Power ≥ 1), the video number is wrong and must be re-recorded.
3. If the cell is at T2+ for a 4-4-4 / 6-6 / 8-4 split, the number is unchanged — keep it.
4. If you used a different cost, deck, or turn than what's tabulated here, compute it in the v0.4 tool — the math is now correct end-to-end.
